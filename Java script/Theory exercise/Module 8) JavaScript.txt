 Question 2: Describe how switch statements work in JavaScript. When should you use a
switch statement instead of if-else?
ANS:A switch statement is used to compare one value against multiple possible cases.
It checks the value, matches it with a case, and executes the code for that case.

Syntax:
switch(expression) {
    case value1:
        // code
        break;

    case value2:
        // code
        break;

    default:
        // code runs when no case matches
}

Example:
let day = 3;

switch(day) {
    case 1:
        console.log("Monday");
        break;
    case 2:
        console.log("Tuesday");
        break;
    case 3:
        console.log("Wednesday");
        break;
    default:
        console.log("Invalid day");
}

Explanation:

day is compared to each case

When day == 3, it prints "Wednesday"

break stops the switch from running other cases

default runs if no case matches

When should you use a switch instead of if-else?

Use a switch when:

You have many conditions to check for the same value

Each condition is a fixed value (1, 2, 3â€¦ or strings like "red", "blue")

Examples of good use cases for switch:

Menu selection

Checking days of the week

Checking user roles (admin, user, guest)

Choosing operations based on a number or string


Loops (For, While, Do-While)
Question 1: Explain the different types of loops in JavaScript (for, while, do-while). Provide abasic
example of each.
ANS:Loops are used to repeat a block of code multiple times until a condition becomes false.
JavaScript has three main loops: for, while, and do-while.

1. for Loop

Used when you know how many times you want to run the loop.

Syntax:
for (initialization; condition; increment) {
    // code to run
}

Example:
for (let i = 1; i <= 5; i++) {
    console.log(i);
}


Output: 1 2 3 4 5

2. while Loop

Used when you donâ€™t know the exact number of iterations and want to keep looping while a condition is true.

Syntax:
while (condition) {
    // code to run
}

Example:
let i = 1;
while (i <= 5) {
    console.log(i);
    i++;
}


Output: 1 2 3 4 5

3. do-while Loop

Always runs at least once, even if the condition is false.

Condition is checked after running the loop.

Syntax:
do {
    // code to run
} while (condition);

Example:
let i = 1;
do {
    console.log(i);
    i++;
} while (i <= 5);


Output: 1 2 3 4 5

Question 2: What is the difference between a while loop and a do-while loop?
ANS:
1. Condition Check

while loop checks the condition before running the loop.

do-while loop checks the condition after running the loop.

2. Execution

while loop may run 0 times if the condition is false at the beginning.

do-while loop runs at least once, even if the condition is false.

3. Syntax Difference

while loop:

while (condition) {
    // code
}


do-while loop:

do {
    // code
} while (condition);

Example:
while loop:
let x = 5;
while (x < 5) {   // false â†’ loop does NOT run
    console.log(x);
}

do-while loop:
let y = 5;
do {
    console.log(y);  // runs once
} while (y < 5);

Question 2: What is the difference between a function declaration and a function
expression?
ANS:1. Function Declaration

A function declaration defines a function using the function keyword with a name.
It is hoisted, meaning it can be called before it is defined in the code.

Example:

function greet() {
    console.log("Hello!");
}

greet();  // can be called before or after definition

2. Function Expression

A function expression stores a function inside a variable.
It is not hoisted, so it can be called only after it is defined.

Example:

const greet = function() {
    console.log("Hello!");
};

greet();  // can only be called after definition

ANS:1. Parameters

Parameters are variables listed in a function definition.
They are used to receive input values when the function is called.

Example:

    console.log(a * b);
}

2. Return Values

A return value is the value that a function sends back to the place where it was called.
The return keyword is used to return a value from a function.

Example:

    return x + y;  // returns result
}

let result = add(10, 5);

JavaScript Error Handling

Question 1: What is error handling in JavaScript? Explain the try, catch, and finally
blocks with an example.
ANS:Error handling in JavaScript is a way to catch and manage runtime errors so that the program does not crash.
It helps the program run smoothly even if something goes wrong.

JavaScript uses the try...catch...finally blocks for error handling.

1. try Block

Contains the code that may cause an error.

2. catch Block

Runs if an error occurs in the try block.

It receives the error information.

3. finally Block

Runs always, whether an error occurs or not.

Used for cleanup code.

Example:
try {
  let result = 10 / 0;
  console.log(result);
} catch (error) {
  console.log("An error occurred:", error.message);
} finally {
  console.log("This will always execute");
}

Output:
Infinity
This will always execute


(If there was a real error, the catch block would run.)

Another Example with Real Error
try {
  let x = y + 10;  // y is not defined â†’ error
} catch (error) {
  console.log("Error:", error.message);
} finally {
  console.log("Program finished");
}


Question 2: Why is error handling important in JavaScript applications?
ANS:Error handling is important because it helps a JavaScript application run smoothly and safely even when unexpected problems occur.

Main reasons why error handling is important:

Prevents program crashes
It stops the entire application from stopping when an error occurs.

Improves user experience
Instead of showing a broken page, friendly error messages can be shown.

Helps in debugging
It provides useful error messages that help developers find and fix problems.

Maintains application stability
Keeps the app running even if part of the code fails.

Improves security
Stops sensitive error details from being shown to users.
